Layer 1 — Physical
Multiple L1 Segments →
One Broadcast Domain
A physical segment is just a chunk of wire (or Wi-Fi). On its own, a segment is an island. When you use a hub or a repeater to link islands together, all those segments merge into one big broadcast domain — because a hub simply shouts everything to everyone, with no filtering.
📢
AnalogyOne hub = one megaphone handed to everyone on the floor. Every PC hears every message, no matter which wire it's plugged into.

Layer 2 — Data Link
Multiple L2 Segments →
Bigger Broadcast Domain
At Layer 2, devices know about MAC addresses. A bridge connects two L2 segments — it's smarter than a hub (it learns MAC addresses) but it still lets broadcasts pass through. So bridging two L2 segments creates one larger broadcast domain.
🌉
AnalogyTwo neighbourhoods connected by an open bridge. A normal message only goes to the right house. But a town-wide announcement (broadcast) crosses the bridge and reaches BOTH neighbourhoods.

Every Device, Placed in Context
Each device operates at a specific layer and has a different effect on broadcast domains.
🔊

Hub (L1)

No brain. Repeats every bit to every port. All ports = same collision domain AND same broadcast domain. Like a megaphone.

🌉

Bridge (L2)

Learns MAC addresses. Splits collision domains but passes broadcasts — so it merges broadcast domains. The ancestor of the switch.

🔀

Switch (L2)

A multi-port bridge. Each port = own collision domain. Smart delivery via MAC table. Still passes broadcasts by default → one broadcast domain per switch (unless VLANed).

🚧

Router (L3)

Works with IP addresses. Blocks all broadcasts — each router interface is a separate broadcast domain. The wall between networks.

🏷️

VLAN (L2 software)

Splits one physical switch into multiple virtual switches, each with its own broadcast domain. No extra hardware needed.

🌐

VXLAN (L2 over L3)

Tunnels an entire virtual L2 network over L3/internet. Creates a broadcast domain that spans multiple physical sites or data centres.

Core Concept
The Broadcast Domain — Who Hears the Shout?
When a device sends a broadcast (e.g. an ARP "Who has IP x.x.x.x?"), the broadcast domain defines how far that shout travels. Every device in the domain must wake up and process it — even if the message isn't for them.
🏘️
Town-Crier analogyImagine a town crier shouting an announcement. Everyone inside the town walls hears it. The WALLS of the town = broadcast domain boundary. Routers are the walls. Hubs/switches are buildings inside the walls.

Virtual LAN
VLAN — Slicing One Switch into Many
Without VLANs, one switch = one broadcast domain. With VLANs, you tag each port with a VLAN ID, and the switch treats each VLAN as if it were a completely separate physical switch with its own broadcast domain.
🏢
Office floor analogyOne big open floor (switch). Management installs invisible soundproof dividers (VLANs). Now HR, Engineering, and Finance each have their own "room." A shout in HR's room doesn't reach Engineering — same floor, different broadcast domains.

Virtual Extensible LAN
VXLAN — Stretching L2 Across the Internet
VLAN is limited to one physical site (~4,094 segments). Cloud data centres need millions of isolated networks across many physical locations. VXLAN solves this by encapsulating entire L2 frames inside UDP packets — tunnelling the local network through the internet.
📦
Courier analogyImagine your local internal mail (L2 frame) being stuffed into a courier envelope (UDP packet) and shipped across cities. When it arrives, it's unwrapped and behaves exactly like local mail again. The cities can be anywhere — the sender/receiver think they're next door.

The Full Picture
How It All Connects
Every concept in this guide is just a different answer to one question: "Who can hear this message?"
Concept Relationships
Hub (L1) → repeats to all ports → same collision AND broadcast domain
Bridge (L2) → learns MACs, splits collision domains but merges broadcast domains
Switch (L2) → multi-port bridge · all ports = one broadcast domain by default
VLAN (L2 virtual) → software slices one switch into N switches · each VLAN = own broadcast domain
Router (L3) → blocks all broadcasts · each interface = separate broadcast domain · connects VLANs
VXLAN (L2 over L3) → tunnels virtual L2 over L3 · creates a broadcast domain spanning multiple sites
Broadcast Domain → the result of all the above — determined by where hubs/bridges/switches merge traffic and where routers/VLANs cut it
